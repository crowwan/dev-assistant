## 작성 가이드

### 제목 작성 팁

| 유형 | 패턴 | 예시 |
|------|------|------|
| 삽질기 | "~삽질기 (그리고 ~한 이유)" | "Docker + EKS 삽질기 (그리고 EC2로 돌아온 이유)" |
| 자동화 | "~, 직접 하고 있나요?" | "매일 아침 스탠드업, 직접 쓰고 있나요?" |
| 개념 | "~가 필요한 이유" | "프론트엔드에서 도메인 모델링이 필요한 이유" |

### 부제목 작성 팁

- 50자 이내
- 독자 공감 포인트 또는 결과 힌트
- 약간의 위트 가능

### 이모지 헤딩 규칙

```
🎯 시작/목표
🐳 1일차/첫 시도 (Docker 관련 아니어도 사용 가능)
☸️ 2일차/두 번째 시도
🤯 현타/전환점
🔄 방향 전환
🤔 고민/왜 필요한가
💻 구현/코드
📌 핵심 포인트
🔧 해결 방법/도구
⚠️ 주의사항/삽질
💡 배운 것들/인사이트
🛠️ 현재 아키텍처
🚀 다음 단계/로드맵
📚 교훈/참고자료
🎉 결과/마무리
❓ 질문/개념 정의
❗ 해결책/중요 포인트
📄 결론
```

### 코드 블록 규칙

- 언어 명시: ```typescript, ```bash, ```yaml
- 주석으로 핵심 포인트 강조
- 코드 전후에 설명 추가
