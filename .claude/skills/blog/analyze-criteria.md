# 블로그 소재 분석 기준

이 문서는 Claude Code 세션을 분석하여 블로그 소재 가치를 판단하는 기준입니다.

---

## 1. 점수화 시스템 (10점 만점)

### 흥미도 (0-3점)
독자가 공감할 만한 상황인가?

| 점수 | 기준 |
|------|------|
| 3 | 많은 개발자가 겪는 보편적 문제/상황 |
| 2 | 특정 분야 개발자가 겪는 문제 |
| 1 | 개인적 상황이지만 일부 공감 가능 |
| 0 | 너무 특수하거나 회사 기밀 관련 |

### 학습가치 (0-3점)
다른 개발자에게 도움되는 내용인가?

| 점수 | 기준 |
|------|------|
| 3 | 재사용 가능한 패턴/도구/방법론 공유 |
| 2 | 특정 기술의 실용적 사용법 공유 |
| 1 | 개인 경험에서 얻은 인사이트 |
| 0 | 학습 가치 없음 |

### 스토리 (0-2점)
시행착오/여정이 있는가?

| 점수 | 기준 |
|------|------|
| 2 | 명확한 시작-문제-해결 흐름, 방향 전환 포함 |
| 1 | 문제-해결 흐름은 있으나 단순함 |
| 0 | 스토리 없이 단순 구현/설정 |

### 적용가능성 (0-2점)
따라할 수 있는 내용인가?

| 점수 | 기준 |
|------|------|
| 2 | 코드/설정 예시 포함, 바로 적용 가능 |
| 1 | 개념은 적용 가능하나 구체적 예시 부족 |
| 0 | 특정 환경에 종속되어 적용 어려움 |

### 총점 해석

- **7점 이상**: 블로그 소재로 적합
- **4-6점**: 조건부 적합 (각색/보강 필요)
- **3점 이하**: 부적합

## 2. 세션 유형 분류

### 블로그 감이 되는 세션

| 유형 | 설명 | 키워드/패턴 |
|------|------|-------------|
| **삽질/회고** | 실패 후 방향 전환, 오버엔지니어링 경험 | "포기", "결국", "삽질", 2회 이상 접근법 변경 |
| **자동화** | 반복 작업을 자동화한 경험 | "자동화", "스킬", "스크립트", Slack/Jira 연동 |
| **새 기술 적용** | TDD, DDD, 패턴 적용 경험 | "TDD", "리팩토링", "도메인 모델", "패턴" |
| **문제 해결** | 복잡한 버그 수정, 디버깅 경험 | "원인", "해결", "디버깅", 에러 메시지 포함 |
| **아키텍처 결정** | 설계 선택, 트레이드오프 경험 | "설계", "아키텍처", "vs", 옵션 비교 |

### 블로그 감이 아닌 세션

| 유형 | 설명 |
|------|------|
| 단순 설정 | 린트 수정, 패키지 업데이트, 환경 설정 |
| 문서 작성 | 문서만 작성한 세션 |
| 짧은 QA | 프롬프트 3개 미만의 짧은 질의응답 |
| 비즈니스 로직 | 회사 기밀 관련 비즈니스 로직 구현 |
| 반복 작업 | 비슷한 작업을 여러 파일에 반복 적용 |

## 3. 프롬프트 패턴 분석

### 소재 가치가 높은 프롬프트 패턴

```
# 삽질 인정 패턴
"이것도 안 되네", "포기하고", "결국", "다시 시작"

# 방향 전환 패턴
"다른 방법으로", "대신에", "vs", "비교해서"

# 문제 해결 패턴
"왜 안 되지", "에러:", "원인", "해결", "디버깅"

# 자동화 패턴
"자동화", "스킬", "스크립트", "반복되는"

# 학습 패턴
"어떻게 하면", "베스트 프랙티스", "패턴"
```

### 소재 가치가 낮은 프롬프트 패턴

```
# 단순 요청
"~해줘", "수정해줘", "추가해줘" (컨텍스트 없이)

# 설정 관련
".gitignore", "린트", "prettier", "config"

# 문서 작성
"README", "문서 작성", "주석 추가"
```

## 5. 추가 고려사항

### 회사 기밀 필터링

다음 키워드가 포함된 세션은 자동 제외:
- 프로젝트명/제품명 직접 언급
- API 키, 토큰, 비밀번호 노출
- 내부 시스템 아키텍처 상세
- 고객 데이터 관련 내용

### 중복 방지

이미 작성된 초안(`./blog-posts/`)과 주제가 겹치면 알림:
- "이미 유사한 주제의 초안이 있습니다: {파일명}"

### 시리즈 가능성

연관된 세션이 여러 개 있으면 시리즈 제안:
- "연속된 3개 세션에서 E2E 테스트 관련 작업 → 시리즈 글 가능"
