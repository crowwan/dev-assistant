## 성과 표현 패턴

### 패턴 1: [동사] + [대상] + [방법] + [결과]

**구조**: 무엇을 어떻게 해서 어떤 결과를 냈는지

**예시**:
- "레거시 시스템을 마이크로서비스로 전환하여 배포 주기를 2주→1일로 단축"
- "정산 쿼리를 Batch Fetching으로 최적화하여 처리 시간 80% 감소"
- "의존성 캐싱을 통해 CI 시간 70% 단축"
- "적절한 index 및 로컬 캐싱을 통한 보험 통계 데이터 응답 속도 최적화"

### 패턴 2: [문제 상황]에서 [해결책]으로 [성과]

**구조**: 어떤 문제 상황에서 무엇으로 해결했는지

**예시**:
- "일 2,000만 PV 트래픽 환경에서 캐시 레이어를 도입하여 DB 부하 70% 감소"
- "SPOF가 될 수 있는 단점을 극복하고, 추상화를 통한 서비스 조직의 생산성 향상"
- "잦은 기획 내용 변경에 테스트코드 작성을 통해 수월하게 대응"

### 패턴 3: [역할]로서 [범위]의 [결과물]

**구조**: 어떤 역할로 어떤 범위를 담당했는지

**예시**:
- "테크 리드로서 5명 규모 팀의 아키텍처 설계 및 코드 리뷰 담당"
- "프로젝트 전체 리드로 참여하여 일정 및 스프린트 관리"
- "BE 개발자로서 아래와 같은 업무를 담당하였습니다"

### 패턴 4: [기술/도구]를 이용한 [대상] [행위]

**구조**: 어떤 기술로 무엇을 했는지

**예시**:
- "Kotlin Coroutines을 이용한 제휴 보험사 API 병렬 및 비동기 호출 설계 및 구현"
- "NGrinder를 이용한 부하 테스트를 통해 각 pod이 안정적으로 처리 가능한 TPS 산정"
- "Jest 기반의 단위 테스트, Docker, Localstack 기반의 통합 테스트 환경 구성"
- "Canvas API 기반의 대용량 랜덤 이미지 생성 및 블록체인 업로드 기능 개발"

### 패턴 5: 약 [숫자]개의 [대상] 작성, 이를 통한 [효과]

**구조**: 정량적 작업량과 그 효과

**예시**:
- "약 180개의 단위 및 통합테스트 코드 작성, 이를 통한 QA 이슈 발생 최소화"
- "약 70개의 단위 및 통합테스트 코드 작성, 이를 통한 BE QA 이슈 0건 발생"
- "50개 이상의 유닛 테스트 작성을 통한 데이터 정합성 검증"

## 자기소개 패턴

### 오프닝 문장

**패턴 A: 연차 + 규모 + 대표 성과**
- "N년차 [직군]으로 [규모 수치]의 [도메인]에서 [대표 성과]를 경험했습니다."
- 예: "11년차 개발자로 회원수 130만의 에듀테크, MAU 1,500만 & 일 주문 300만 커머스, 일 PV 2,000만 포탈 등의 서비스에서 백엔드 플랫폼 개발 및 AWS 인프라 구축 등을 해왔습니다."

**패턴 B: 가치관 + 강점**
- "[가치관]을 중요시하며 [강점]을 가지고 있습니다."
- 예: "서비스의 안정성과 유지보수성을 최우선으로 고려하며, 이를 프로덕트에 녹여 냅니다."

**패턴 C: 도전 + 역할**
- "새로운 [무엇]을 좋아하는 [직군] [이름]입니다."
- 예: "새로운 도전을 좋아하는 소프트웨어 엔지니어 정규호입니다."

### 역량 나열

**패턴**: "[기술/환경]에서의 [역할/결과물] 경험/구성"

**예시**:
- "TS 환경에서의 백엔드 아키텍처, 자동화된 테스트 환경 등 구성"
- "Java, SpringBoot, Spring Batch, Spring Cloud를 활용한 엔터프라이즈 애플리케이션 구현"
- "PostgreSQL, MySQL 등에서 수억 ~ 수십억건에서의 쿼리튜닝, 인덱스 설계"
- "AWS 클라우드 환경에서의 CI/CD, 애플리케이션 서버 환경등 구성"

## 프로젝트 설명 패턴

### 배경/목표

- "[비즈니스 문제]를 해결하기 위한 [프로젝트명]"
- "[기존 시스템]의 [한계]를 극복하기 위해 시작"
- "[서비스명]은 [기능]을 제공합니다"

**예시**:
- "배달의 민족 정산 시스템을 온프레미스 인프라에서 AWS 클라우드로 마이그레이션 프로젝트"
- "기존 SP(Stored Procedure)로만 이루어진 포인트 환경을 AWS 기반의 마이크로 서비스 환경으로 전환"

### 기여/역할

- "프로젝트 전체 리드로 참여하여 [범위] 담당"
- "[기술]을 도입하여 [문제] 해결"
- "[대상]을 쉽게 [행위]할 수 있도록 [방법] 설계"

**예시**:
- "시스템 전체 도메인 모델링"
- "코드리뷰 및 아키텍처 리뷰"
- "외부 API를 쉽게 호출할 수 있도록 추상화"

### 결과/성과

- "[서비스명]는 [시점]에 오픈한 이래 안정적으로 운영되고 있습니다"
- "[방법]을 통해 [지표] [N]% [개선동사]"
- "덕분에 [긍정적 결과]"

**예시**:
- "건강관리 서비스는 12월에 오픈한 이래 안정적으로 운영되고 있습니다"
- "덕분에 외부 요인에 의한 장애 외에는 서비스 운영 중 장애가 발생하지 않았습니다"

## 경력 설명 패턴

### 담당 업무 서술

**패턴**: "[도메인/서비스]를 개발 및 운영하였습니다"

**예시**:
- "마이데이터 보험, 보험 비교추천, 내차관리 서비스를 개발 및 운영하였습니다"
- "포털 메인 페이지, 회원 서비스의 메인 개발자로 개발/운영"

### 성과 서술

**패턴**: "[행위]를 통해 [효과]"

**예시**:
- "기술적 부채를 주도적으로 맡아 처리하며, 지속 가능한 서비스가 되도록 꾸준히 관리하였습니다"
- "지식과 경험을 기술 블로그 글 기고 등을 통해 팀 내외 동료들과 공유하였습니다"
