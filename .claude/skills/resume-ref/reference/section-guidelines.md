# 섹션별 이력서 작성 가이드

> Rallit 인기 이력서 분석 기반 작성 가이드

## 1. 기본 정보 섹션

### 필수 항목
- 직군 (예: 백엔드 개발자, 프론트엔드 개발자)
- 이름
- 이메일
- 연차

### 한줄 소개 (핵심)
**공식**: `N년차 [직군]으로 [핵심 도메인/규모]에서 [대표 성과]를 경험했습니다.`

**좋은 예시**:
- "11년차 개발자로 회원수 130만의 에듀테크, MAU 1,500만 & 일 주문 300만 커머스, 일 PV 2,000만 포털 등의 서비스에서 백엔드 플랫폼 개발 및 AWS 인프라 구축 등을 해왔습니다."

**나쁜 예시**:
- "백엔드 개발자입니다." (구체성 없음)
- "열심히 일하는 개발자입니다." (성과 없음)

---

## 2. 자기소개 섹션

### 구조
1. **오프닝**: 핵심 역할/가치관 한 문장
2. **역량 나열**: 불릿 포인트로 3~5개
3. **차별점**: 다른 개발자와 다른 점

### 역량 나열 패턴
```markdown
- [기술/환경]에서의 [역할/결과물] 경험
- [도구]를 활용한 [목표] 달성
- [상황]에서 [행동]을 통한 [역량] 보유
```

**좋은 예시**:
- "TS 환경에서의 백엔드 아키텍처, 자동화된 테스트 환경 등 구성"
- "Java, SpringBoot, Spring Batch, Spring Cloud 를 활용한 엔터프라이즈 애플리케이션 구현"
- "PostgreSQL, MySQL 등에서 수억 ~ 수십억건에서의 쿼리튜닝, 인덱스 설계"

---

## 3. 기술 스택 섹션

### 구성
- **주력**: 가장 자신있는 기술 (3~5개)
- **사용 가능**: 프로젝트에서 사용해본 기술

### 배치 순서
1. 언어 (Java, TypeScript, Python)
2. 프레임워크 (Spring Boot, NestJS, React)
3. 데이터베이스 (MySQL, PostgreSQL, MongoDB)
4. 인프라/도구 (AWS, Docker, Kubernetes)

---

## 4. 경력 섹션

### 구조
```markdown
### [회사명] | [직책] | [팀명]
YYYY.MM - YYYY.MM (N년 N개월)

[회사/팀의 역할 한줄 설명]
- [성과 1]
- [성과 2]
- [성과 3]
```

### 성과 작성 원칙
1. **정량화**: 수치를 반드시 포함
2. **기술적 디테일**: 어떤 기술로 어떻게 해결했는지
3. **비즈니스 임팩트**: 비즈니스에 어떤 영향을 미쳤는지

**좋은 예시**:
- "사내 모니터링 & 알람 도입 (APM / CloudWatch & Lambda)"
- "PostgreSQL 성능 튜닝 및 장애 대응"
- "개발파트 채용 브랜딩 개선 (백엔드 / 프론트엔드)"

---

## 5. 프로젝트 섹션

### 구조
```markdown
### [프로젝트명]
[회사명]
YYYY.MM - YYYY.MM (N개월)

[프로젝트 배경/목표 설명]
- [기여 1]
- [기여 2]
- [성과]
```

### 작성 포인트
1. **Why**: 왜 이 프로젝트가 필요했는지
2. **What**: 무엇을 했는지 (역할)
3. **How**: 어떻게 해결했는지 (기술)
4. **Result**: 어떤 결과가 나왔는지 (성과)

**좋은 예시**:
```markdown
### 배달의 민족 정산 시스템 온프레미스 → AWS 마이그레이션
우아한형제들
2020.06 - 2020.11 (6개월)

배달의 민족 정산 시스템을 온프레미스 인프라에서 AWS 클라우드로 마이그레이션 프로젝트
- 클라우드 배포 환경, 애플리케이션 클라우드 인프라 환경 구성
- 데이터베이스 이관 지원
- 주요 API, Batch 성능 테스트
- 클라우드 환경에 맞는 애플리케이션 코드 변경
```

## 6. 포트폴리오 섹션

### 추천 항목
1. **GitHub**: 코드 품질 증명
2. **기술 블로그**: 기술적 깊이 증명
3. **발표 자료**: 커뮤니케이션 능력 증명
4. **오픈소스 기여**: 협업 능력 증명

### 링크 제공 팁
- 단순 URL보다 설명 추가
- 대표 글/프로젝트 하이라이트

---

## 7. 교육 섹션

### 구조
```markdown
### [학교/기관명]
[학위/과정] | [전공]
YYYY.MM - YYYY.MM
```

### 포함 항목
- 대학교 (전공)
- 부트캠프/교육과정 (관련 있는 경우)
- 자격증 (관련 있는 경우)

---

## 체크리스트

### 작성 전
- [ ] 경력 기간 정확히 계산
- [ ] 각 회사/프로젝트별 주요 성과 정리
- [ ] 정량적 수치 확보 (처리량, 개선율 등)

### 작성 후
- [ ] 약한 동사 → 강한 동사 교체
- [ ] 정량적 성과 포함 여부 확인
- [ ] 문장 패턴 일관성 확인
- [ ] 오탈자/문법 검토
