## 약한 표현 → 강한 표현 변환

### 개발/구현

| 약한 표현 | 강한 표현 | 사용 맥락 |
|----------|----------|----------|
| 만들었다 | **설계**, **구축**, **개발**, **구현** | 새로운 것을 만들 때 |
| 했다 | **주도**, **리드**, **담당**, **수행** | 역할을 설명할 때 |
| 고쳤다 | **개선**, **최적화**, **리팩토링** | 기존 것을 고칠 때 |
| 바꿨다 | **전환**, **마이그레이션**, **현대화** | 기술 변경 시 |
| 썼다 | **작성**, **기고**, **문서화** | 문서/글 작성 시 |

| 문제 해결 | **해소**, **극복**, **대응** | "기술 부채 해소" |

## 상황별 추천 동사 리스트

### 신규 개발

```
설계하다 / 구축하다 / 개발하다 / 구현하다 / 도입하다 / 반영하다
```

**예시 문장**:
- "Kafka event publish, consume 구조 **설계**"
- "Jest 기반 테스트 환경 **구축**"
- "인터렉션 라이브러리 **개발** 및 유지보수"

### 기존 시스템 개선

```
개선하다 / 최적화하다 / 리팩토링하다 / 고도화하다 / 현대화하다
```

**예시 문장**:
- "쿼리 **최적화**를 통해 처리 시간 80% 감소"
- "대규모 공용 모듈 **리팩토링**으로 기술 부채 해소"
- "보험 도메인 설계를 **고도화**"

### 마이그레이션/전환

```
전환하다 / 이관하다 / 마이그레이션하다 / 분리하다
```

**예시 문장**:
- "온프레미스에서 AWS 클라우드로 **마이그레이션**"
- "SP 방식에서 API 호출 방식으로 **전환**"
- "메인 DB에서 전용 DBMS로 **분리**"

### 운영/관리

```
운영하다 / 관리하다 / 유지보수하다 / 모니터링하다
```

**예시 문장**:
- "100개 이상의 서비스에서 사용하는 디자인 시스템을 **유지보수**"
- "서비스 **모니터링** 개선"

### 문서화/공유

```
문서화하다 / 공유하다 / 발표하다 / 기고하다
```

**예시 문장**:
- "팀 내외 동료들과 **공유**"
- "기술 블로그 글 **기고**"
- "선행조사, **문서화** 및 공유"

## 동사 강도 레벨

### Level 1 (기본)
만들다, 하다, 쓰다, 고치다

### Level 2 (일반)
개발하다, 구현하다, 작성하다, 수정하다

### Level 3 (강한) ✅ 추천
설계하다, 구축하다, 최적화하다, 주도하다, 리드하다

### Level 4 (임팩트) ⭐ 최고
달성하다, 혁신하다, 선도하다, 확립하다

## 피해야 할 표현

| 표현 | 문제점 | 대안 |
|------|--------|------|
| ~했습니다 (단순) | 구체성 없음 | 구체적 행위 동사 사용 |
| 노력했습니다 | 결과 없음 | 달성한 결과로 대체 |
| 도와드렸습니다 | 수동적 | "담당", "수행" |
| 배웠습니다 | 성과 아님 | 적용한 결과로 대체 |
| 참고했습니다 | 성과 아님 | "도입", "반영" |
