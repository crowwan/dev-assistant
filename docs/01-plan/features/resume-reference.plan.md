# Plan: Resume Reference (이력서 자동 생성 스킬)

> 좋은 이력서 패턴을 사전 분석하고, 사용자의 세션/커밋/기존 이력서를 기반으로 실제 이력서를 md 형식으로 자동 생성하는 Claude 스킬

## 배경 및 문제

### 현재 상황
- 이력서 작성 시 어떤 문장/표현을 써야 할지 막막함
- 내가 한 일은 알지만 "이력서용 문장"으로 바꾸기 어려움
- 좋은 이력서의 문장 패턴을 배우려면 직접 분석해야 함

### 문제점
1. **문장력 부족**: 기술적인 내용을 임팩트 있게 표현하기 어려움
2. **수동 작업**: 커밋/세션 기록을 일일이 정리해서 이력서에 반영해야 함
3. **일관성 부족**: 매번 다른 스타일로 작성하게 됨

## 목표

1. **분석 문서**: 좋은 이력서의 문장 패턴/표현 방식을 사전 정리
2. **자동 생성**: 내 데이터(세션, 커밋, 기존 이력서)를 분석해서 이력서 md 생성
3. **문장 품질**: 임팩트 있는 이력서용 문장으로 자동 변환

## 핵심 기능

### 1. 사전 분석 문서 (Reference Docs)

AI가 Rallit 등에서 분석한 내용을 문서로 정리:

```
.claude/skills/resume-ref/
├── reference/
│   ├── sentence-patterns.md    # 문장 패턴 모음
│   ├── action-verbs.md         # 동사/표현 사전
│   ├── metrics-examples.md     # 정량적 성과 표현법
│   └── section-guidelines.md   # 섹션별 작성 가이드
```

#### sentence-patterns.md 예시
```markdown
## 성과 표현 패턴

### 패턴 1: [동사] + [대상] + [방법] + [결과]
- "레거시 시스템을 마이크로서비스로 전환하여 배포 주기를 2주→1일로 단축"
- "정산 시스템 쿼리를 최적화하여 처리 시간 80% 감소"

### 패턴 2: [문제 상황]에서 [해결책]으로 [성과 달성]
- "일 2,000만 PV 트래픽 환경에서 캐시 레이어를 도입하여 DB 부하 70% 감소"

### 패턴 3: [역할]로서 [범위]의 [결과물] 담당
- "테크 리드로서 5명 규모 팀의 아키텍처 설계 및 코드 리뷰 담당"

## 자기소개 문장 패턴

### 오프닝
- "N년차 [직군]으로 [핵심 도메인]에서 [핵심 성과]를 경험했습니다."
- "[가치관/철학]을 중요시하며 [구체적 행동]을 실천합니다."

### 역량 나열
- "[기술/도구]를 활용한 [결과물] 구축 경험"
- "[상황]에서 [역할]을 수행하며 [역량] 보유"
```

#### action-verbs.md 예시
```markdown
## 이력서용 동사 사전

### 개발/구현
| 약한 표현 | 강한 표현 |
|----------|----------|
| 만들었다 | 설계/구축/개발/구현 |
| 했다 | 주도/리드/담당 |
| 고쳤다 | 개선/최적화/리팩토링 |

### 성과/결과
| 상황 | 추천 동사 |
|------|----------|
| 속도 개선 | 단축, 감소, 최적화 |
| 규모 확장 | 확장, 스케일업, 증가 |
| 품질 향상 | 개선, 강화, 안정화 |

### 협업/리더십
- 조율, 협업, 가이드, 멘토링, 온보딩
```

### 2. 이력서 자동 생성

#### 입력 소스
1. **Claude 세션 기록**: `~/.claude/sessions/` 분석
2. **Git 커밋 히스토리**: 프로젝트별 커밋 메시지 분석
3. **기존 이력서**: 사용자가 제공한 이전 이력서
4. **decisions 폴더**: `~/.claude/decisions/` 의사결정 기록

#### 생성 프로세스
```
[입력 데이터 수집]
    ↓
[주요 성과/프로젝트 추출]
    ↓
[reference 문서 참조하여 문장 변환]
    ↓
[md 형식 이력서 출력]
```

## 스킬 인터페이스

```bash
# 참고 문서 보기
/resume-ref patterns          # 문장 패턴 보기
/resume-ref verbs             # 동사 사전 보기

# 이력서 생성
/resume-ref generate          # 대화형으로 이력서 생성
/resume-ref generate --from-sessions   # 세션 기록 기반 생성
/resume-ref generate --from-git        # Git 히스토리 기반 생성

# 기존 이력서 개선
/resume-ref improve path/to/resume.md  # 문장 개선 제안

# 특정 섹션만 생성
/resume-ref write intro       # 자기소개 생성
/resume-ref write experience  # 경력 섹션 생성
/resume-ref write project     # 프로젝트 섹션 생성
```

## 병렬 분석 아키텍처

### 서브에이전트 병렬 실행

여러 이력서를 동시에 분석하기 위해 서브에이전트를 병렬로 실행:

```
[메인 Claude]
    │
    ├──→ [서브에이전트 1] → 이력서 A 분석 → 패턴 추출
    ├──→ [서브에이전트 2] → 이력서 B 분석 → 패턴 추출
    ├──→ [서브에이전트 3] → 이력서 C 분석 → 패턴 추출
    └──→ [서브에이전트 4] → 이력서 D 분석 → 패턴 추출
                │
                ↓
        [결과 취합 & 공통 패턴 도출]
                │
                ↓
        [Reference 문서 생성]
```

### 서브에이전트 역할

| 에이전트 | 역할 | 출력 |
|---------|------|------|
| `resume-analyzer` | 개별 이력서 분석 | 문장 패턴, 표현, 구조 추출 |
| `pattern-merger` | 결과 취합 | 공통 패턴 도출, 중복 제거 |

### 분석 프롬프트 (서브에이전트용)

```markdown
## 이력서 분석 태스크

다음 이력서를 분석하여 추출해주세요:

1. **문장 패턴**: 성과를 표현하는 문장 구조
   - 예: "[동사] + [대상] + [방법] + [결과]"

2. **강한 동사/표현**: 임팩트 있는 동사와 표현
   - 예: "구축", "최적화", "주도"

3. **정량적 성과 표현**: 수치를 포함한 성과 문장
   - 예: "처리 시간 80% 단축", "MAU 100만 달성"

4. **섹션별 특징**: 자기소개, 경력, 프로젝트 각 섹션의 특징

출력 형식: JSON
```

## 구현 계획

### Phase 1: Reference 문서 작성 (병렬 분석)
1. Rallit 인기 이력서 URL 수집 (5~10개)
2. **서브에이전트 병렬 실행**하여 각 이력서 분석
3. 결과 취합하여 `reference/` 폴더에 문서 작성
   - `sentence-patterns.md` - 문장 패턴
   - `action-verbs.md` - 동사/표현 사전
   - `metrics-examples.md` - 정량적 성과 표현
   - `section-guidelines.md` - 섹션별 가이드

### Phase 2: 기본 생성 기능
1. `SKILL.md` 작성
2. 대화형 이력서 생성 (`/resume-ref generate`)
3. reference 문서 참조하여 문장 품질 향상

### Phase 3: 자동 데이터 수집
1. Claude 세션 기록 분석 (`--from-sessions`)
2. Git 커밋 히스토리 분석 (`--from-git`)
3. decisions 폴더 분석

### Phase 4: 문장 개선 기능
1. 기존 이력서 분석
2. 약한 표현 → 강한 표현 변환 제안
3. 정량적 성과 추가 제안

## 출력 예시

### 입력 (커밋 메시지들)
```
- fix: 정산 쿼리 N+1 문제 해결
- feat: Redis 캐시 레이어 추가
- refactor: 주문 시스템 모듈화
```

### 출력 (이력서 문장)
```markdown
## 프로젝트: 정산 시스템 성능 최적화

### 성과
- 정산 쿼리의 N+1 문제를 Batch Fetching으로 해결하여 API 응답 시간 60% 단축
- Redis 캐시 레이어를 도입하여 DB 부하 감소 및 처리량 3배 향상
- 주문 시스템을 모듈화하여 코드 재사용성 향상 및 테스트 커버리지 80% 달성
```

## 성공 기준

1. `/resume-ref generate` 실행 시 실제 사용 가능한 이력서 md 출력
2. reference 문서의 패턴을 적용하여 임팩트 있는 문장 생성
3. 사용자 데이터(세션/커밋) 기반으로 개인화된 내용 생성

## 의존성

- 필수: `~/.claude/sessions/` (세션 기반 생성 시)
- 선택: Git (커밋 기반 생성 시)

## 리스크

1. **데이터 부족**: 대화형으로 직접 입력 받아 해결
2. **문장 품질**: reference 문서를 지속 개선
3. **개인화 부족**: 여러 소스 조합으로 해결
