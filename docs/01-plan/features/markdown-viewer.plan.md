# Plan: Markdown Viewer

> 클로드 코드에서 생성한 마크다운 문서를 터미널 내에서 바로 볼 수 있는 뷰어 기능

## 배경 및 문제

### 현재 상황
- 클로드 코드가 문서(보고서, 계획서 등)를 생성하면 파일 경로만 출력됨
- 사용자가 직접 파일을 열어서 확인해야 함 (VS Code, cat 등)
- 터미널 워크플로우가 중단됨

### 문제점
1. **컨텍스트 전환**: 터미널 → 에디터 → 터미널 이동 필요
2. **불편함**: 단순히 내용 확인만 하려는데 에디터를 열어야 함
3. **일관성 부족**: 클로드 코드 내에서 모든 작업을 완료할 수 없음

## 목표

1. 터미널에서 마크다운 문서를 보기 좋게 렌더링
2. `/view` 스킬로 간단하게 호출
3. 최근 생성된 문서를 빠르게 열 수 있는 단축 기능

## 기능 범위

### MVP (최소 기능)
- [ ] 파일 경로로 마크다운 뷰어 실행
- [ ] 터미널에서 마크다운 렌더링 (glow, bat, mdcat 등 활용)
- [ ] `/view <path>` 스킬 구현

### 추가 기능 (선택)
- [ ] `/view --last` - 마지막으로 생성된 문서 열기
- [ ] `/view --list` - 최근 문서 목록 표시
- [ ] 페이지네이션 (긴 문서 처리)

## 기술 선택지

### 옵션 1: glow 사용 (추천)
- **장점**: 터미널 마크다운 렌더링 전문 도구, 예쁜 출력
- **설치**: `brew install glow`
- **사용**: `glow path/to/file.md`

### 옵션 2: bat 사용
- **장점**: 이미 많이 설치되어 있음, 신택스 하이라이팅
- **설치**: `brew install bat`
- **사용**: `bat path/to/file.md`

### 옵션 3: mdcat 사용
- **장점**: 이미지 렌더링 지원 (iTerm2)
- **설치**: `brew install mdcat`
- **사용**: `mdcat path/to/file.md`

### 옵션 4: 순수 터미널 출력
- **장점**: 추가 도구 불필요
- **단점**: 렌더링 품질 낮음

## 스킬 인터페이스

```bash
# 기본 사용법
/view docs/report.md

# 마지막 생성 문서
/view --last

# 최근 문서 목록
/view --list

# 특정 도구 지정
/view docs/report.md --tool glow
```

## 구현 계획
