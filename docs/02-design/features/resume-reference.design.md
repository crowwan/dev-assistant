# Design: Resume Reference (이력서 자동 생성 스킬)

> 좋은 이력서 패턴을 병렬 분석하고, 사용자 데이터 기반으로 이력서를 자동 생성하는 스킬

## 참조 문서
- Plan: `docs/01-plan/features/resume-reference.plan.md`

---

## 1. 스킬 구조

### 파일 위치
```
.claude/skills/
└── resume-ref/
    ├── SKILL.md                    # 스킬 정의
    └── reference/
        ├── sentence-patterns.md    # 문장 패턴 모음
        ├── action-verbs.md         # 동사/표현 사전
        ├── metrics-examples.md     # 정량적 성과 표현
        └── section-guidelines.md   # 섹션별 가이드
```

### 스킬 메타데이터
```yaml
---
name: resume-ref
description: 이력서 문장 패턴 참고 및 자동 생성
---
```

## 2. 명령어 인터페이스

| 명령어 | 설명 | 예시 |
|--------|------|------|
| `/resume-ref analyze` | Rallit 이력서 병렬 분석 (Reference 생성) | `/resume-ref analyze` |
| `/resume-ref patterns` | 문장 패턴 보기 | `/resume-ref patterns` |
| `/resume-ref verbs` | 동사 사전 보기 | `/resume-ref verbs` |
| `/resume-ref generate` | 대화형 이력서 생성 | `/resume-ref generate` |
| `/resume-ref generate --from-git` | Git 기반 생성 | `/resume-ref generate --from-git` |
| `/resume-ref improve <path>` | 기존 이력서 개선 | `/resume-ref improve resume.md` |

## 3. 병렬 분석 아키텍처

### 3.1 전체 흐름

```
┌─────────────────────────────────────────────────────────────────┐
│ Step 1: Rallit 이력서 URL 수집                                    │
│   - 인기순 정렬된 개발자 이력서 5~10개 선정                          │
│   - 백엔드/프론트엔드/풀스택 다양하게 포함                           │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 서브에이전트 병렬 실행                                     │
│                                                                 │
│   [메인 Claude] ─┬─→ [researcher #1] → 이력서 A 분석              │
│                 ├─→ [researcher #2] → 이력서 B 분석              │
│                 ├─→ [researcher #3] → 이력서 C 분석              │
│                 ├─→ [researcher #4] → 이력서 D 분석              │
│                 └─→ [researcher #5] → 이력서 E 분석              │
│                                                                 │
│   각 에이전트: WebFetch로 페이지 읽기 → JSON 형식으로 패턴 추출      │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: 결과 취합 & Reference 문서 생성                            │
│   - 공통 패턴 추출                                                │
│   - 중복 제거                                                    │
│   - reference/ 폴더에 마크다운 파일 생성                           │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 서브에이전트 Task 호출

```javascript
// 병렬 Task 호출 (단일 메시지에 여러 Tool 호출)
Task(subagent_type="researcher", prompt=`
  이력서 분석 태스크

  URL: ${url}

  다음을 추출해서 JSON으로 반환:
  1. sentence_patterns: 성과 표현 문장 구조
  2. strong_verbs: 임팩트 있는 동사/표현
  3. metrics: 정량적 성과 문장
  4. section_features: 섹션별 특징
`, run_in_background=true)
```

### 3.3 분석 결과 스키마

```typescript
interface ResumeAnalysis {
  resume_id: string;
  name: string;
  role: string;  // "백엔드", "프론트엔드", "풀스택"
  years: number; // 연차

  patterns: {
    intro: string[];      // 자기소개 패턴
    experience: string[]; // 경력 패턴
    project: string[];    // 프로젝트 패턴
  };

  strong_verbs: string[];  // ["구축", "최적화", "주도"]
  metrics: string[];       // ["MAU 1,500만", "응답시간 60% 단축"]

  raw_sentences: {
    intro: string[];       // 실제 자기소개 문장들
    experience: string[];  // 실제 경력 문장들
    project: string[];     // 실제 프로젝트 문장들
  };
}
```

## 4. Reference 문서 설계

### 4.1 sentence-patterns.md

```markdown
# 이력서 문장 패턴

> Rallit 인기 이력서 N개 분석 결과 (생성일: YYYY-MM-DD)

## 성과 표현 패턴

### 패턴 1: [동사] + [대상] + [방법] + [결과]
**구조**: 무엇을 어떻게 해서 어떤 결과를 냈는지
**예시**:
- "레거시 시스템을 마이크로서비스로 전환하여 배포 주기를 2주→1일로 단축"
- "정산 쿼리를 Batch Fetching으로 최적화하여 처리 시간 80% 감소"

### 패턴 2: [문제 상황]에서 [해결책]으로 [성과]
**구조**: 어떤 문제 상황에서 무엇으로 해결했는지
**예시**:
- "일 2,000만 PV 트래픽 환경에서 캐시 레이어를 도입하여 DB 부하 70% 감소"

### 패턴 3: [역할]로서 [범위]의 [결과물]
**구조**: 어떤 역할로 어떤 범위를 담당했는지
**예시**:
- "테크 리드로서 5명 규모 팀의 아키텍처 설계 및 코드 리뷰 담당"

## 자기소개 패턴

### 오프닝 문장
- "N년차 [직군]으로 [핵심 도메인]에서 [핵심 성과]를 경험했습니다."
- "[가치관]을 중요시하며 [구체적 행동]을 실천합니다."

### 역량 나열
- "[기술]를 활용한 [결과물] 구축 경험"
- "[상황]에서 [역할]을 수행하며 [역량] 보유"

## 프로젝트 설명 패턴

### 배경/목표
- "[비즈니스 문제]를 해결하기 위한 [프로젝트명]"
- "[기존 시스템]의 [한계]를 극복하기 위해 시작"

### 기여/역할
- "프로젝트 전체 리드로 참여하여 [범위] 담당"
- "[기술]을 도입하여 [문제] 해결"
```

### 4.2 action-verbs.md

```markdown
# 이력서용 동사/표현 사전

## 개발/구현

| 약한 표현 | 강한 표현 | 사용 맥락 |
|----------|----------|----------|
| 만들었다 | 설계, 구축, 개발, 구현 | 새로운 것을 만들 때 |
| 했다 | 주도, 리드, 담당, 수행 | 역할을 설명할 때 |
| 고쳤다 | 개선, 최적화, 리팩토링 | 기존 것을 고칠 때 |
| 바꿨다 | 전환, 마이그레이션, 현대화 | 기술 변경 시 |

## 성과/결과

| 상황 | 추천 동사 | 예시 |
|------|----------|------|
| 속도 개선 | 단축, 감소, 최적화 | "응답 시간 60% 단축" |
| 규모 확장 | 확장, 스케일업, 증가 | "처리량 3배 확장" |
| 품질 향상 | 개선, 강화, 안정화 | "가용성 99.9% 달성" |
| 비용 절감 | 절감, 감소, 효율화 | "인프라 비용 40% 절감" |

## 협업/리더십

| 역할 | 추천 표현 |
|------|----------|
| 팀 리딩 | 리드, 이끌다, 조율 |
| 지식 전파 | 멘토링, 온보딩, 가이드 |
| 협업 | 협업, 조율, 커뮤니케이션 |
```

### 4.3 metrics-examples.md

```markdown
# 정량적 성과 표현 가이드

## 트래픽/규모

| 패턴 | 예시 |
|------|------|
| 일/월 단위 | "일 PV 2,000만", "MAU 1,500만" |
| 동시 접속 | "동시 접속자 10만 처리" |
| 데이터 규모 | "일 100GB 데이터 처리" |

## 성능 개선

| 패턴 | 예시 |
|------|------|
| 퍼센트 감소 | "응답 시간 60% 단축", "에러율 90% 감소" |
| 배수 향상 | "처리량 3배 향상", "속도 10배 개선" |
| 절대값 | "평균 응답 시간 200ms → 50ms" |

## 품질/안정성

| 패턴 | 예시 |
|------|------|
| 커버리지 | "테스트 커버리지 80% 달성" |
| 가용성 | "서비스 가용성 99.9% 유지" |
| 장애 대응 | "장애 복구 시간 1시간 → 10분" |

## 팀/조직

| 패턴 | 예시 |
|------|------|
| 팀 규모 | "5명 규모 팀 리드" |
| 프로젝트 기간 | "6개월간 프로젝트 리드" |
| 채용/온보딩 | "신규 입사자 10명 온보딩" |
```

```
│   Q: 경력 연차는? (1~3년/4~7년/8년 이상)                          │

                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ Step 2: 경력 정보 수집                                           │

│   Q: 주요 성과나 프로젝트가 있다면?                                │
│   (자유 형식 입력 → AI가 구조화)                                  │

                              ↓
┌─────────────────────────────────────────────────────────────────┐
│ Step 3: Reference 문서 참조하여 문장 생성                          │

                              ↓
┌─────────────────────────────────────────────────────────────────┐

```

## 6. 출력 형식

### 6.1 생성된 이력서 (md)

```markdown
# [이름] - [직군]

## 기본 정보
- **이메일**: your@email.com
- **GitHub**: https://github.com/username
- **블로그**: https://your-blog.com

> 7년차 백엔드 개발자로 MAU 1,500만 규모의 커머스 플랫폼에서
> 결제/정산 시스템을 설계하고 운영한 경험이 있습니다.

## 자기소개

안정적인 시스템 설계와 팀 성장을 중요시하는 개발자입니다.

- Java, Spring Boot 기반 대규모 트래픽 처리 경험
- 레거시 시스템 현대화 및 마이크로서비스 전환 주도
- 테스트 코드 작성과 코드 리뷰를 통한 품질 문화 정착

## 기술 스택

**주력**: Java, Spring Boot, JPA, MySQL, Redis, AWS
**사용 가능**: TypeScript, Node.js, Docker, Kubernetes

## 경력

### 인프랩 | CTO | 개발팀
2021.04 - 현재 (4년)

제품 조직과 시스템의 성장이 비즈니스의 성과가 될 수 있도록 관리/설계합니다.

- 사내 모니터링 & 알람 체계 구축 (APM / CloudWatch & Lambda)
- 신규 서비스 전체 아키텍처 및 도메인 모델 설계
- PostgreSQL 성능 튜닝 및 장애 대응으로 서비스 안정성 99.9% 달성
- 개발 문화 및 채용 브랜딩 개선 (백엔드/프론트엔드)

## 프로젝트

### 정산 시스템 성능 최적화
2023.01 - 2023.06 (6개월)

일 처리량 100만 건 규모의 정산 시스템 성능 개선 프로젝트

- 정산 쿼리의 N+1 문제를 Batch Fetching으로 해결하여 API 응답 시간 60% 단축
- Redis 캐시 레이어를 도입하여 DB 부하 감소 및 처리량 3배 향상
- 비동기 처리 도입으로 정산 완료 시간 2시간 → 30분으로 단축
```

### 6.2 개선 제안 출력

```markdown
## 이력서 개선 제안

### 1. 자기소개 섹션

| 현재 | 개선안 | 이유 |
|------|--------|------|
| "로그인 기능을 개발했습니다" | "JWT 기반 인증 시스템을 설계/구축하여 보안성 강화" | 약한 동사 → 강한 동사 |

### 2. 경력 섹션

💡 **정량적 성과 추가 권장**
- "API 개발" → "일 100만 요청 처리하는 API 개발"
- "성능 개선" → "응답 시간 200ms → 50ms로 75% 개선"

### 3. 프로젝트 섹션

✅ 잘 작성된 부분:
- "처리 시간 80% 감소" - 정량적 성과 포함

⚠️ 개선 필요:
- "여러 기능을 추가함" → 구체적인 기능명과 성과 명시
```

## 7. 에러 처리

| 상황 | 처리 |
|------|------|
| Reference 문서 없음 | `/resume-ref analyze` 실행 안내 |
| Git 히스토리 없음 | 대화형 생성으로 fallback |
| 이력서 파일 없음 | 에러 메시지 + 생성 안내 |
| 분석 실패 (Rallit 접근 불가) | 캐시된 Reference 사용 |
